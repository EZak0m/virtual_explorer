# 🚀 Virtual Explorer 開発ロードマップ (Updated)

## 🛠️ 現在の実装状況

### 核心機能
- [x] **仮想座標システム**: チャンク単位での移動と探索済み情報の保持。
- [x] **バイオーム表示とカスタム色設定 (New!)**: GUI右下のグリッドツールチップに、チャンク・ブロック座標に加え、その座標のバイオームを表示する機能を追加。また、Config（BIOME_COLORS）で指定されたカラーで多言語表示名に色付けされる仕組みを実装完了。
- [x] **FE/液体/アイテムの自動搬出入**: 隣接チェスト、パイプ、電池への対応。
  - [x] **FEの強制自動搬入**: `canExtract()`を無視して隣接ブロックからエネルギーを「強制消去」して受電するロジック（リフレクション対応）を完備。
- [x] **地図連携**: 地図の中心へのワープ、探検家の地図によるロックオン、宝の地図 of 高速探索。
- [x] **アップグレードシステム**: 21億FE対応、内部データ保存、圧縮アップグレード。
- [x] **カスタムモデル**: Blockbenchによる詳細な3Dモデル（スチームパンク風デスク）の適用。

### アップグレード要素
- [x] **スピードアップグレード**: 探索速度上昇（最大64個）。FE消費倍率: $1.01^{Count}$。
- [x] **並列処理コア**: 1回の工程での出力数増加（最大64個）。FE消費倍率: $1.1^{Count}$。
- [x] **製図家のコンパス (Recycle)**: 地図等の消費確率を軽減（最大5枚に制限）。FE消費倍率: $1.01^{Count}$。
- [x] **流体サンプラー**: 溶岩や水などの液体資源回収。FE消費倍率: $1.0^{Count}$。
- [x] **地上/地下/ネザー/エンドモジュール**: ターゲットとなる環境を定義する基本モジュール。

### 固有アップグレード (各種1枚まで搭載可能)
以下の「強力な特殊機能」は、スロットの過負荷を防ぐため**各種1枚ずつのみインストール可能**（異なる種類であれば同時にすべて搭載可能）とします。

*   **発掘調査の刷毛 (Archeology)**
    *   効果：砂漠の寺院や海底遺跡などの「発掘」に特化。
    *   FE消費倍率：$1.0$ (影響なし)
*   **精密採取 (Silk Touch)**
    *   効果：ブロックをそのままの形で回収（幸運と競合）。
    *   FE消費倍率：$2.0^{Count}$
*   **無限の可能性 (Infinite)**
    *   効果：同じ場所を無限に探索し続ける（座標が動かない）。
    *   FE消費倍率：$10.0^{Count}$
*   **構造物への興味 (Structure Interest)**
    *   効果：構造物の発見と滞在探索を有効化。
    *   FE消費倍率：$5.0^{Count}$
*   **解体業者モジュール (Demolition)**
    *   効果：周辺ブロック（石、土など）をバルク回収。
    *   FE消費倍率：$5.0^{Count}$

## ✅ 最終チェックリスト

### 1. インストール・搭載機能の正常性
- [x] アップグレードを手に持って「Shift + 右クリック」することで、仮想探索機に直接インストールできることを確認（アイテムの `useOn` オーバーライド）。
- [x] 固有アップグレードが同じ種類であれば1枚のみしか入らず、別種であればそれぞれ1枚ずつ同時搭載できることを確認。
- [x] 製図家のコンパスが最大5枚を超えてインストールされないよう制限（`canAddUpgrade`）。
- [x] 競合するアップグレード（例: 幸運と精密採取）を搭載しようとした際、後に搭載したアップグレードを取り込まず「§c競合するアップグレードが装着されています」等のエラーを表示。

### 2. FE搬入の強制消去ロジック
- [x] 隣接ブロックにFEが貯蔵されている場合、`canExtract()` を無視してリフレクションを用いて `energy` フィールドを直接操作し、強制消去と同時に自機バッファへ自動搬入する仕組みを完備。

### 3. バランス調整と計算式
- [x] 消費電力の上昇倍率計算を `(基礎値) × (アップグレード1枚当たりの上昇倍率) ^ (アップグレードの枚数)` となるように指数関数（Math.pow）へ刷新。
- [x] スピードアップグレード（1.01倍）、並列処理コア（1.1倍）、コンパス（1.01倍、上限5枚）、流体（1倍）、構造物（5倍）、精密（2倍）、解体（5倍）、無限（10倍）の倍率が掛け合わされることを確認。

### 4. バイオーム表示とカスタムカラー設定 (New!)
- [x] GUI右下のグリッドツールチップに「Biome: [名前]」が表示され、Minecraft側の日本語などの多言語表示名が正しく取得されていることを確認。
- [x] Configの `biome_colors` （Visualカテゴリ内）で指定されたカラー（Hexコード `#RRGGBB`）を正しくロード・パースし、テキスト表示の色付けに反映できていることを確認。
